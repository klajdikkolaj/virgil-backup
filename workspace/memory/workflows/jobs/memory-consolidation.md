# Job SOP - Memory Consolidation

## Objective
Consolidate volatile daily memory into durable, low-token long-term memory.

## Steps
1. Read recent `memory/YYYY-MM-DD.md` files (focus last 7 days).
2. Extract only durable items:
- stable preferences
- repeated constraints
- important decisions
- open verification debts
3. Update long-term files:
- `MEMORY.md`
- `memory/preferences.md`
- project/contact files if needed
4. Keep memory lean:
- remove stale or duplicate entries
- avoid raw logs in long-term files

## Promotion rule
Promote only if at least 2 apply:
- reused >= 3 times in 14 days
- high consequence if forgotten
- stable >= 7 days
- explicitly confirmed by Klajdi

## Output
Return concise status with:
- files updated
- key promotions
- key removals
- unresolved uncertainties
